\section{Algorithm Description}

\subsection{Problem Formulation}

\subsubsection{State Representation}
A state consists of:
\begin{itemize}
    \item 8 cascades
    \item 4 free cells
    \item 4 foundations
\end{itemize}

\subsubsection{Actions}
Valid actions include:
\begin{itemize}
    \item Move between cascades
    \item Move to a free cell
    \item Move to a foundation
\end{itemize}

\subsubsection{Goal Test}
The goal state is achieved when all cards are moved to the foundations in order from Ace to King.

\subsubsection{Cost Function}
Each move has a cost of:
\[
cost = 1
\]

\subsubsection{Heuristic}
A simple heuristic:
\[
h(n) = \text{number of cards not yet in foundations}
\]

\subsubsection{Avoiding Revisiting States}
Use a \texttt{visited} set to store explored states.

% =========================
\subsection{Algorithm Details}

\subsubsection{Breadth-First Search (BFS)}
Uses a queue data structure, exploring level by level. 
Guarantees finding the optimal solution.

\subsubsection{Depth-First Search (DFS) / IDS}
DFS uses a stack or recursion. IDS incrementally increases depth to improve upon DFS.

\subsubsection{Uniform-Cost Search (UCS)}
Uses a priority queue, selecting the node with the lowest cost.

\subsubsection{A* Search}
Formula:
\[
f(n) = g(n) + h(n)
\]
Where:
\begin{itemize}
    \item $g(n)$: cost from start to current node
    \item $h(n)$: heuristic estimate
\end{itemize}
